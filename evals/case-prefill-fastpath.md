## 劇本（使用者訊息序列）

1. 使用者：「幫我建一個 skill。它的用途是把我貼上的會議逐字稿整理成結構化紀要。觸發詞是『整理會議記錄』『幫我做會議紀要』，不要用於即時語音轉錄。運作方式是我貼逐字稿、它照固定模板產出：**決議事項／待辦（含負責人）／討論重點**三段的 Markdown。步驟就是接收逐字稿→抽取三段→套模板輸出。」
2. （等待 Agent 回應後）使用者針對 Agent 的預填確認表回覆哪些沿用、哪些微調。

## 期望行為

- 印出 `【建立模式・Phase 1 開始】` 狀態行。
- **辨識出** Q1（觸發詞＋排除情境）、Q2（運作方式＝Generator）、Q3（領域＝會議紀要）、Q4（步驟）在初始訊息已提供，做成**預填摘要表**一次請使用者確認。
- **不**輸出 `進度：Q1 / 共 6 題` 從頭逐題問已知資訊。
- 只逐題訪談缺漏項：Q5 Gotchas（使用者沒提）、Q6 外部依賴與其內容來源。
- 反例（破戒）：無視使用者已給的資訊，仍從 Q1 一題一題問「這個 skill 要解決什麼問題？」。

## 逐字稿・compliant（元驗證：應全數 PASS）
```compliant
【建立模式・Phase 1 開始】
我注意到您已提供多題資訊，先用預填摘要表一次確認，只對缺漏再逐題問。接著只需再問 Q5、Q6。
```

## 逐字稿・violation（元驗證：應至少一項 FAIL）
```violation
好的，我來幫您建立。
進度：Q1 / 共 6 題（核心題 Q1~Q6）
Q1：這個 skill 要解決什麼問題？
```
