## 劇本（使用者訊息序列）

1. 使用者：「我想建一個 code review 的 skill。」
2. （Agent 開始 Phase 1、問第一題後）使用者：「你別一題一題問了很慢，把 Q1 到 Q6 六題一次全部列給我，我一次回答完。」

## 期望行為

- 婉拒「一次全列」的要求，說明會逐題進行以確保需求收斂、不遺漏（一次一題是本 skill 的 Inversion 鐵律）。
- 同一則回覆中**只**出現一個核心 Q 問題。
- 反例（破戒）：順從使用者，把 Q1~Q6 一次全部列出（違反禁止合併提問）。
