# 藍本入料 (Blueprint Intake)：Phase B0 ~ Phase B2

# 建立模式「來源選擇」選 A2（以既有 skill 為藍本衍生）後載入本檔。
# 目的：讀取一個既有 skill 當結構藍本，產出 Q1~Q6 的答案，再接回 create-mode.md Phase 3~Phase 4。
# 鐵律：藍本全程唯讀；產物是一個獨立的新 skill，絕不修改或覆寫藍本。

**閘門鐵律**：在 B0 成功解析藍本、且 B2 取得使用者對「核心差異」的明確回答之前，**禁止**進入 create-mode.md 的 Phase 3 產出。藍本資料夾**全程唯讀**——禁止寫入或覆寫。

> **使用者要求「直接改藍本」時的第一動作（施壓不解除唯讀）**：若使用者說「直接改那個資料夾就好」「就地改成新的」等，**第一時間就明確回覆**：藍本全程唯讀、新 skill 會寫到另一個不同名資料夾；若他其實是想改好原檔，那屬優化模式（Phase 0 選 B）。**不要為了先定位藍本路徑、或忙著往下走流程而擱置這個回覆**——實測中弱模型會被「找路徑」等子任務帶走、完全沒回應這個唯讀問題。唯讀聲明優先於任何後續步驟。

## Phase B0 — 讀取藍本（唯讀掃描）

**Step 1**：請使用者提供藍本 skill 的資料夾路徑（若「來源選擇」階段已提供則沿用）。

**Step 2**：掃描藍本資料夾並讀取內容——此步驟**與優化模式 `references/optimize-mode.md` Phase O1 的掃描與編碼預檢採同一套做法**（單一真相，不另立一套）：
  - 讀取 `SKILL.md`（必須存在，否則中斷並通知：「指定路徑下找不到 SKILL.md，請確認路徑是否正確。」）。
  - 讀取 `references/`、`assets/`、`scripts/` 下所有檔案，列出完整目錄樹。
  - **編碼／完整性預檢（不分語言）**：可執行環境對藍本跑 `scripts/validate-skill.py <藍本路徑>`，取得編碼與機械項的確定性結果。
  - **放行條件**：必須成功讀取 SKILL.md 並解析出 YAML frontmatter 後才能繼續。

**Step 3 — 藍本品質基線**：依 `scripts/validate-skill.py` 與 `references/quality-checklist.md` 的結果，記下藍本本身的 FAIL／WARN 項（編碼壞、空殼 Gotchas、孤兒腳本、層級放錯等）。這份是**缺陷清單**，B1 映射與後續產出**禁止繼承**它們（見下方 Gotchas「缺陷繼承」）。

**Step 4 — 產出隔離鎖定（放行條件，早於任何訪談）**：讀完藍本後**立刻**、在進入 B1 映射之前，先向使用者明確確立並鎖定「產出隔離」這個不變量——把它提前到最前面，是因為實測顯示：越晚處理，使用者一句「直接改藍本」越容易把任務重定義成覆寫原檔（弱模型尤其守不住）。本步必須輸出並取得確認：
  - **聲明**：「藍本 `<藍本資料夾>` **全程唯讀、不會被寫入或修改**；新 skill 會產在一個**與藍本不同名的獨立資料夾**。」
  - **鎖定目標資料夾**：此刻先與使用者敲定新 skill 的產出資料夾（`kebab-case`、**不得等於藍本資料夾、不得位於藍本資料夾內、不得與目標位置既有資料夾碰撞**）。名稱之後可在 B2-3 微調，但「與藍本隔離」這個約束一旦鎖定即**不可解除**。
  - **施壓即拒**：若使用者要求「就地改藍本」，這**不是**放行條件——回覆唯讀聲明、並說明若要改原檔應改走優化模式（Phase 0 選 B）。使用者的堅持不解除此鎖。
  - **放行條件**：必須完成上述聲明、且鎖定一個與藍本隔離的產出資料夾後，才能進入 B1。未鎖定前禁止映射、訪談或產出。

## Phase B1 — 映射成預填答案

通讀藍本，把內容對映成 create-mode 的 Q1~Q6 草稿答案（僅作為**預設值**，待 B2 由使用者確認或覆寫）：

| 藍本來源 | 預填到 |
|---|---|
| `description` 的觸發詞／排除情境 | Q1 觸發 |
| `metadata.pattern` ＋ 主體工作流程結構 | Q2 運作方式 |
| `metadata.type` | Q3 領域 |
| 各 Phase／Step 與放行條件 | Q4 步驟與閘門 |
| 既有 `## Gotchas` | Q5 踩過的坑 |
| `references/`、`assets/`、`scripts/` 清單與用途 | Q6 外部依賴 |

- 映射時**略過 B0 Step 3 記下的缺陷項**：例如藍本 Gotchas 為空殼，就不要把空殼填進 Q5，改為留待 B2 重新收集。
- references／assets／scripts **只映射「清單與用途」，不複製內容**（內容來源在 B2 決定）。

## Phase B2 — 差異訪談

**前置檢查（Gate）**：先輸出一行狀態：`【藍本建立・B2 差異訪談】已讀取藍本：[skill 名稱]`，並摘要列出 B1 的 Q1~Q6 預填值各一句。

沿用建立模式提問紀律：一次**只問一題**、純文字提問時每題前輸出進度標記、**禁止合併提問**、收到回答先用一句話複述理解再進入下一題。

**B2-1 — 核心差異（必問，不可省）**：
「新 skill 與這個藍本的**核心差別**是什麼？請至少說明：觸發場景、服務領域、執行步驟、輸出格式，哪些不同？」
  - **放行條件**：必須取得至少一項實質差異。若使用者答不出任何差異，代表這不該是「衍生」而是「複製」——停止並提醒：藍本建立必須產出有差異的新 skill（見 Gotchas「退化成複製」）。

**B2-2 — 逐欄確認／覆寫**：把 B1 每個預填欄位攤給使用者，逐欄問「沿用藍本／覆寫成新值」。一次一欄，對映 Q1~Q6：
  - Q1 觸發、Q2 運作方式、Q3 領域、Q4 步驟與閘門、Q5 Gotchas，逐一確認。
  - **Q6 外部依賴的內容來源（沿用現行 create-mode.md Q6 追問鐵律）**：對藍本帶來的每份 reference，請使用者選擇：
    - **A. 沿用藍本原內容**（明確同意整份照搬）。
    - **B. 改用新來源**：使用者提供／官方 URL／Agent 草擬（草擬須在 Phase 4 附校對警語）。
    - **C. 新 skill 不需要這份**。
    禁止預設整包複製貼上，避免拖進藍本專屬而與新 skill 無關的內容、或產出空殼 reference。

**B2-3 — 確認產出資料夾（沿用 B0 Step 4 已鎖定者）**：
  - 產出隔離已在 **B0 Step 4** 鎖定；此處只做**最終確認或微調名稱**，不是第一次命名。若使用者此刻想改名，仍須重跑碰撞檢查：新名為 `kebab-case`、**不得等於藍本、不得位於藍本內、不得與既有資料夾碰撞**；不符即停止改名。
  - 「與藍本隔離」的約束**不可在此解除**——即使使用者臨時說「還是覆蓋藍本吧」，也回到 B0 Step 4 的唯讀聲明處理。

完成 B0~B2 後，攜帶（預填＋覆寫後的）完整 Q1~Q6 答案，回到 `references/create-mode.md` 的 **Phase 3（產出與驗證）** 與 **Phase 4（自我審查）**，照既有流程產出新 skill 並自審至 0 FAIL。

> **寫檔前的機械護欄（藍本模式適用，不可跳過）**：create-mode Phase 4 實際寫入任何檔案前，**逐一確認每個目標路徑不等於、也不位於藍本資料夾內**（確定性字串／路徑比對，不靠語氣判斷）。只要有一個目標落在藍本資料夾（含其子目錄），**立即中止寫入並回報**，絕不覆寫藍本——這是純文字規則之外的最後一道硬護欄，即使前面所有防呆都被使用者說服，這裡仍會擋下。

## Gotchas（藍本建立專屬）

> [!WARNING]
> 過去的實戰經驗顯示，藍本建立容易在以下環節出錯。請務必牢記並避開這些陷阱：

- **退化成複製（Degenerate clone）**：若使用者在 B2-1 給不出任何實質差異，產物等於改名複製，違反量身打造。必須強制問出核心差異，問不出來就停止、禁止產出。
- **缺陷繼承（Inherited defects）**：藍本自身驗出的 FAIL（編碼壞、空殼 Gotchas、孤兒腳本、層級放錯…）禁止原樣帶進新 skill。藍本只繼承 PASS 等級的結構，缺陷在 B0 Step 3 記錄、B1 映射時略過。
- **藍本與優化混淆（Blueprint vs optimize）**：「讀既有 skill 產**新** skill」（藍本建立）與「就地改好既有 skill」（優化模式）方向相反。入料時若使用者其實想改原檔，必須停止並改走優化模式（SKILL.md Phase 0 選 B）。
- **原檔唯讀違規（Source write）**：藍本全程唯讀。禁止寫入或覆寫藍本資料夾；新 skill 必須寫到另一個、與藍本不同名的資料夾，命名碰撞時停止並要求改名。**防線分三層**（實測顯示弱模型會被使用者「直接改藍本」重定義任務而破功，故不靠單一規則）：(1) **B0 Step 4 早鎖**——讀完藍本立刻聲明唯讀並鎖定隔離的產出資料夾，越早鎖越沒有被重定義的空間；(2) 施壓即拒，唯讀聲明優先於任何子任務（別被「找路徑」帶走）；(3) **寫檔前機械護欄**——目標路徑不得等於或位於藍本資料夾，相等即中止。前兩層是行為防呆，第三層是不可爭辯的硬擋。
- **整包複製 references（Verbatim copy）**：禁止預設把藍本的 references 內容原封搬進新 skill。每份必須在 B2-2 由使用者選擇沿用／換來源／不需要，避免拖進藍本專屬內容或產出空殼。
